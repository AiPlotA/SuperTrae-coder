# Plan CEO Review 技能（产品 5 维度 review）

> 基于 gstack `/plan-ceo-review` Skill 改造
> 目的：找到 spec 中"描述的是 5-star 产品,实际只算 3-star 工具"的维度
> 强制级别：P1 - spec 阶段必走

---

## 5 维度 0-10 评分模板

### 维度 1：愿景（Vision）—— 我们要成为什么？

| 分 | 长什么样 |
|----|---------|
| 0 | "做一个工具" / "做个 demo" |
| 3 | 有功能描述但无愿景 |
| 5 | 有愿景但不可测量（"让世界更美好"）|
| 7 | 有清晰愿景 + 1 年 / 3 年里程碑 |
| **10** | **愿景让 1 个陌生人听完 30 秒后能复述"我们要成为 X"** |

### 维度 2：用户（User）—— 谁会用？为什么？

| 分 | 长什么样 |
|----|---------|
| 0 | "所有人" / "开发者" |
| 3 | 有角色但无场景 |
| 5 | 有角色 + 场景,但无具体画像 |
| 7 | 有 3 个具体用户画像（含年龄段 / 收入 / 痛点）|
| **10** | **能说出"如果只能为 1 个用户做,会选谁,为什么"的清晰优先级** |

### 维度 3：价值（Value）—— 用户付钱/时间的理由？

| 分 | 长什么样 |
|----|---------|
| 0 | "好用" / "方便" |
| 3 | 有功能价值但无量化 |
| 5 | 有 ROI 估算但不可验证 |
| 7 | 有 baseline → 改进后的数字对比 |
| **10** | **能说出"用户在 X 场景下,每月节省 Y 小时 / 多赚 Z 元"的硬数字** |

### 维度 4：差异化（Differentiator）—— 凭什么选我们不选竞品？

| 分 | 长什么样 |
|----|---------|
| 0 | 没调研竞品 |
| 3 | 列出竞品但无差异点 |
| 5 | 有 1 个差异点但易被复制 |
| 7 | 有 1-2 个难复制的差异点（数据 / 网络效应 / 锁定）|
| **10** | **有"护城河",竞品要追 12+ 个月,且我们能持续扩展** |

### 维度 5：可执行（Executable）—— 团队能在 6 周内交付吗？

| 分 | 长什么样 |
|----|---------|
| 0 | "做一下试试" |
| 3 | 有时间估算但无依赖关系 |
| 5 | 有 6 周计划 + 关键风险 |
| 7 | 有 MVP → 完整版 2 阶段 + 1 句话"如果时间不够砍什么" |
| **10** | **有"明确不做的 5 件事"且团队能在 6 周内交付 MVP 验证** |

## Fix 输出格式

```markdown
## CEO Review 评分

| 维度 | 当前 | 目标 | 差距 |
|------|------|------|------|
| 愿景 | 4/10 | 8/10 | -4 |
| 用户 | 7/10 | 8/10 | -1 |
| 价值 | 3/10 | 8/10 | -5 |
| 差异化 | 5/10 | 8/10 | -3 |
| 可执行 | 6/10 | 8/10 | -2 |
| **平均** | **5.0** | **8.0** | **-3.0** |

## Top 3 修复方向

1. **维度 3 价值**（-5 最大）：[具体修复方法]
2. **维度 4 差异化**（-3）：[具体修复方法]
3. **维度 1 愿景**（-4）：[具体修复方法]

## 下一步

- 修完 3 个方向后,重跑本 skill
- ≥ 7 分后,自动加载 skill/plan-eng-review
```

## 3 强制问题

### 问题 1：北极星指标是什么？

> 不要接受"DAU" / "用户增长"。追问到具体数字：
> - 基线多少？（例：现在 0 / 现在 100）
> - 3 个月目标多少？（例：1000 / 10000）
> - 怎么测量？

### 问题 2：3 个最难说服的早期用户是谁？

> 不要接受"愿意试的人"。追问到具体人：
> - 他们的名字 / 角色 / 联系方式（可选）
> - 为什么他们会试？
> - 试完后他们会留下还是走？

### 问题 3：如果只能保留 1 个功能,会留哪个？

> 这是 YAGNI 的核心检验：
> - 列 5 个候选功能
> - 选 1 个,说为什么
> - 砍掉的 4 个,放弃它们失去了什么

## 反模式

- ❌ 5 维度都给 8 分"看起来都还行"
- ❌ 用"愿景宏大"为借口不追问可执行
- ❌ 跳过 3 强制问题直接给 fix
- ❌ 把"易用性"当差异化（除非能证明竞品 10× 难用）
