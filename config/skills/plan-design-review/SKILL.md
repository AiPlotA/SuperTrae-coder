# Plan Design Review 技能（设计 4 维度 review）

> 基于 gstack `/plan-design-review` Skill 改造
> 目的：找到 spec 中缺失的 UX/视觉/交互/可访问性维度
> 强制级别：P1 - 任何含 UI 的 spec 必走

---

## 4 维度 0-10 评分模板

### 维度 1：UX（用户体验流程）—— 用户从进入到完成目标的路径清晰吗？

| 分 | 长什么样 |
|----|---------|
| 0 | 无流程图 / 无用户故事地图 |
| 3 | 有 happy path 但无异常路径 |
| 5 | 有完整 happy + 2 个异常路径 |
| 7 | 有 5+ 关键路径 + 每步 ≤3 次点击 |
| **10** | **新用户 5 分钟内能完成核心目标,且过程中没有"为什么这样设计"的疑问** |

### 维度 2：视觉（Visual）—— 设计系统 + 一致性

| 分 | 长什么样 |
|----|---------|
| 0 | 无设计系统 / 颜色随便选 |
| 3 | 有 1-2 套样式但无 token |
| 5 | 有 design token 但应用不一致 |
| 7 | 完整 design token + 8pt 栅格 + 暗色模式 |
| **10** | **设计 token 100% 复用,任意 2 个页面切换无视觉断层** |

### 维度 3：交互（Interaction）—— 反馈 + 状态 + 错误

| 分 | 长什么样 |
|----|---------|
| 0 | 加载无反馈 / 错误无提示 |
| 3 | loading + error 2 态 |
| 5 | loading + success + error + empty 4 态 |
| 7 | 4 态 + 过渡动画 + 撤销支持 |
| **10** | **任意操作都有 ≤200ms 反馈,任意错误都有"做什么能恢复"的指引** |

### 维度 4：可访问性（Accessibility）—— 残障用户能用吗？

| 分 | 长什么样 |
|----|---------|
| 0 | 无 a11y 考虑 |
| 3 | 有 alt + 键盘 tab |
| 5 | 满足 WCAG 2.1 AA |
| 7 | 满足 AAA + 屏幕阅读器测试 |
| **10** | **色盲 / 运动障碍 / 视障用户都能 100% 完成核心目标** |

## 3 强制问题

### 问题 1：核心任务的"5 步关键路径"是什么？

> 不要接受"用户能做 X"。追问到 5 步内：
> - 进入 → 完成目标,5 步以内
> - 每步的"成功信号"是什么
> - 哪步最容易卡住,如何拆

### 问题 2：3 个最常见的"用户会卡住"场景是什么？

> 不要接受"用户会用"。追问具体卡点：
> - 找不到按钮 / 看不懂文案 / 操作不可撤销
> - 每个卡点的"恢复路径"是什么

### 问题 3：色盲 / 键盘 / 屏幕阅读器用户能完成核心任务吗？

> 不要接受"我们用了 aria-label"。追问实测：
> - 键盘 tab 走完核心任务?
> - 色盲模拟器看过主流程?
> - NVDA / VoiceOver 测过?

## 反模式

- ❌ "我们会考虑 a11y"（无具体测试）
- ❌ 设计 token 用了但 50% 页面写死颜色
- ❌ 加载态用 spinner 但成功后不跳转
- ❌ 错误提示"操作失败"不告诉用户"做什么能恢复"
