# 跨工件一致性分析（Cross-Artifact Analysis）

> 基于 spec-kit /analyze + 工程哲学 v1 原则 6 多视角评审
> 目的：发现 spec 与实现之间的偏差

---

## 触发场景

- 用户说"分析"、"一致性"、"对比"、"检查"
- 完成实现后、提交 PR 前
- 发现 spec 与实现疑似不匹配
- 用户问"实现了吗"、"覆盖了吗"

---

## 一致性检查清单

### 1. 规范 → 实现

- [ ] spec 中每个 FR-XXX 都有对应实现
- [ ] 每个 FR 的输入/输出/错误处理都已实现
- [ ] 实现的公开 API 在 spec 中定义
- [ ] 关键实体字段与数据模型一致

### 2. 规范 → 计划

- [ ] 计划任务覆盖所有 FR
- [ ] 任务粒度合理（2-5 分钟）
- [ ] 任务依赖 DAG 正确
- [ ] 关键节点有门控

### 3. 计划 → 实现

- [ ] 每个任务都有代码提交
- [ ] 实现回应了任务描述
- [ ] 没有未跟踪的实现
- [ ] 任务验收标准达成

### 4. 规范 → 测试

- [ ] 每个 FR 有对应测试
- [ ] 每个成功标准（SC-XXX）有测试
- [ ] 边界情况有测试
- [ ] 错误处理有测试

### 5. 跨工件命名

- [ ] 实体名、字段名、API 名一致
- [ ] 无歧义命名
- [ ] 命名符合规范

### 6. 跨工件接口

- [ ] 函数签名一致
- [ ] 返回类型一致
- [ ] 错误码一致
- [ ] 错误信息格式一致

## 一致性分析报告模板

```markdown
# 跨工件一致性分析报告

## 基本信息
- 规范：specs/<feature>-spec.md
- 实现：src/
- 测试：tests/
- 分析时间：YYYY-MM-DD

## 整体评分：X/10

## FR 实现情况

| FR ID | 状态 | 备注 |
|-------|------|------|
| FR-001 | ✅ | 完全实现 |
| FR-002 | ⚠️ | 错误处理未完全实现 |
| FR-003 | ❌ | 完全未实现 |

## 场景覆盖

| 场景 | 测试存在 | 测试通过 | 备注 |
|------|---------|---------|------|
| 场景 1 | ✅ | ✅ | |
| 场景 2 | ⚠️ | - | 缺边界测试 |

## 命名一致性

| 类型 | spec | 实现 | 一致？ |
|------|------|------|--------|
| 实体名 | User | User | ✅ |
| 字段名 | email | email | ✅ |
| API 名 | /users | /api/v1/users | ⚠️ 路径差异 |

## 接口一致性

| 接口 | spec | 实现 | 一致？ |
|------|------|------|--------|
| POST /users | 返回 201 + User | 返回 201 + User | ✅ |
| GET /users/:id | 404 if not found | 500 if not found | ❌ |

## 关键发现

### 缺失
- FR-003 完全未实现
- 场景 2 缺边界测试

### 偏差
- API 路径不一致（spec: /users, 实现: /api/v1/users）
- 错误码不一致（spec: 404, 实现: 500）

## 改进建议

### P0（必须修改）
1. 实现 FR-003
2. 修正 API 路径

### P1（应该修改）
1. 添加场景 2 边界测试
2. 统一错误码

## 结论
- [ ] 通过：实现与规范一致
- [ ] 修订：需要补充实现
- [ ] 否决：重大偏差
```

## 分析方法

### 方法 1：FR 逐项验证

遍历 spec 中每个 FR，在实现中搜索：
- 函数名
- 类名
- API 端点
- 关键逻辑

### 方法 2：场景→测试覆盖矩阵

```
| 场景 | unit test | integration test | e2e test |
|------|-----------|------------------|----------|
| 场景 1 | ✅ | ✅ | ✅ |
| 场景 2 | ⚠️ 缺边界 | ❌ | ❌ |
```

### 方法 3：实体→数据库模型对比

```
| 实体字段 | spec 类型 | DB 类型 | 一致？ |
|---------|----------|---------|--------|
| id | string | uuid | ⚠️ |
| email | string | varchar(255) | ✅ |
```
